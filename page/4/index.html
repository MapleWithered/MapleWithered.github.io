PC和IMP终于搞清楚了！！！

发表于 2019-12-26 更新于 2020-03-09
本文字数： 909 阅读时长 ≈ 1 分钟

我！搞懂！PC和IMP了！

原来法国人也没考虑那么多啊（笑）

分享一下这个nb的方法：

在正常方法很难判断PC/IMP的时候，判断说话人的语气。如果是叙述性的话语，则为IMP；如果是总结性的话语，则为PC。

“叙述”和“总结”这两个词有点抽象，但是有两种简单的情况之前是接触过的：描述背景或状态为“叙述”（IMP），描述动作本身为“总结”（PC），即暗指动作使故事继续向前推进。

有几个例子可以很好地利用这种方法进行判断：

La jeune fille (s’appeler) Cécile Maime. Bien sûr, le prince Ypal et la jeune Cécile Maime (se marier) mais ils (ne pas avoir) beaucoup d’enfants.

此处“名为”为叙述，“结婚”为总结，“孩子不多”为结果性的动作总结。

因此IMP,PC,PC。

Comme c’était le jour de la fête nationale, toutes les rues (être) décorées. Le soir, elle (aller) au bal pour danser et elle (rencontrer) un beau et grand Martiniquais.

此处“街道正在装饰”为叙述（背景描述），“去了舞会”为动作总结性语气，“遇到了”为总结性语气。

Martin (demander) à Constance : « Est-ce que tu veux venir avec moi ? ». Martin (vouloir) vivre une aventure et partir à la découverte du monde qu’il (ne pas connaître).

demander为动作总结（瞬间动词，已发生并完成）（用常规方法可轻松判断），vouloir为叙述，描述Martin之所以这么问的原因，用IMP。ne pas connaître为叙述，即当时不知道的那些知识（判断否定时可以先用肯定形式进行判断，如此处肯定形式中“他所知道的知识”为描述性语气，而非动作总结，因此为IMP）。

参考资料：

《法语复合过去时与未完成过去时的区分》，聂丽娜，《文学教育下半月》2016年第05期。

OpenCV安装历程

发表于 2019-10-21 更新于 2020-03-09
本文字数： 743 阅读时长 ≈ 1 分钟

安装OpenCV-4.1.1时，遇到了一些问题，以下是问题分析和部分解决方案。

缺少Eigen

首先cmake参数是这样的：
cmake -D CMAKE_BUILD_TYPE=RELEASE -D CMAKE_INSTALL_PREFIX=/usr/local/opencv4 -D OPENCV_EXTRA_MODULES_PATH="../opencv_contrib-4.1.1/modules" ../opencv-4.1.1

现象：cmake很正常地Configure过了，然后在make的时候提示找不到Eigen。

原因：Ubuntu中安装Eigen后的include目录在/usr/include/eigen3/Eigen中，而OpenCV会在/usr/local/include/Eigen处寻找

解决：ln -s /usr/include/eigen3/Eigen /usr/local/include/Eigen，创建软链接。

缺少test_common.hpp

现象：make在11%进度，编译opencv_test_dnn_RELEASE.gch时提示fatal error: test_common.hpp: 没有那个文件或目录

原因：未知

解决：将build/../opencv-4.1.1/modules/dnn/test/test_common.hpp复制到build/modules/dnn/test_common.hpp成功继续编译。

缺少scene.h

现象：make在25%进度，编译opencv_test_sfm_RELEASE.gch时提示fatal error: scene.h: 没有那个文件或目录

原因：未知

解决：同上，将相应文件复制到build中的相关目录后可正常编译。

重装的过程依旧很快，很轻松地就完成了（？）。

sct再次安装

官方说是要安装libxrandr和libx11，但是在apt里安装了这两个相应的dev软件包后依然提示缺少include文件。遂想起第一次调教时是先装flux再编译的，于是装了flux再编译sct，顺利完成。

Anyconnect

第一次折腾时疯狂折腾各种library都依然有问题，这次直接在新系统里使用最新安装包一步到位（难道上一次是我脸黑？vmware也不能安装虚拟网卡T^T）

Gnome美化

sudo apt install gnome-tweak-tool，安装Sierra-dark主题，顿时拥有了一个暗黑系风格+右上红绿灯按钮。

安装gnome-shell-extension-dashtodock，获得一个类Mac OS X的dock栏。

安装字体+VSCode美化

将Win中字体全部复制并批量安装，vscode中选用如下配置：

'Consolas', '等线', monospace, 'Droid Sans Fallback'

Ubuntu调教历程-1

发表于 2019-10-04 更新于 2020-03-09
本文字数： 1.6k 阅读时长 ≈ 1 分钟

安装系统

因为宿舍里放了一台台式机作为主力机兼仓库，所以很自然而然地萌生了把手头这台轻薄本装成Linux的想法。由于一直保持着半年重装一次的好习惯（真的是好习惯吗？），所以里面基本上没有放什么数据，于是就直接全盘格掉了x

Ubuntu的版本选择了19.04，拿了一个U盘用UltraISO写入镜像就很快地把系统装好了。（单系统安装起来真的很方便也不用去折腾Grub）

浏览器

装好系统之后第一时间把FireFox卸载了（Chrome是人类之光！），打开Chrome官网下载了deb版的Chrome软件包，遂安装之。

输入法

默认的ibus的超智能（划掉）拼音虽然没有搜狗拼音那种云输入的智能感，但是对于一般用户而言是足够了（而且在系统中集成得也非常好）。

可是！对于一个重度的小鹤双拼，而且是鹤形玩家来说，用这种智能输入法简直就是一场折磨（orz），所以自然是想到了手机端鹤形所基于的iRime同文平台（不同平台的叫法不同）

【中州韻】 ibus-rime → Linux
【小狼毫】 Weasel → Windows
【鼠鬚管】 Squirrel → Mac OS X

把小鹤官方的iRime方案放到了ibus-rime的方案文件夹中按照Readme的说明进行部署，安装中州韻成功√

iBus+rime+小鹤双拼+鹤形 是人类之光！（认真安利脸）

网易云+TIM+各种Wine软件

网易云音乐选择了网易官方的Ubuntu-18.04版本客户端，实测完美运行。

TIM、7-zip、百度网盘、迅雷等软件均使用了Deepin-Wine中已经打包好的版本，安装起来超方便，也均能完美运行（记得把Windows的Font文件夹里的字体都安装过来）

Steam！

官方安装包不解释x

Linux下支持的游戏超多，满足日常使用肯定没有问题！（虽然轻薄本也没什么可以打得动的大游戏T^T）

IDE

VSCode + 教育版JetBrains之CLion

经过一段时间的使用之后的感觉是……CLion好丑……

VSCode又好看又万能！VSCode是人类之光！

（上课记笔记现在都用VSCode+MarkDown了 效率巨高）

F.lux 移植

Windows上一直用Flux来调节屏幕的色温，在Linux上也有相应的Flux版本，但是似乎不是很稳定（在色温改变的时候经常会闪烁）

经过搜索，在Github上找到了一个叫做sct的项目，可以修改屏幕的色温，并且可以通过在屏幕上覆盖类似半透明遮罩（实际是修改Gamma）的方式来调节屏幕的整体“亮度”。

手上的这台HP Elitebook x360 1030 G2的屏幕是PWM调光，在日常亮度下眼睛经常（大约每用5分钟左右）会极度不适（划重点：极度），因此用了sct之后就可以达到一个防止屏幕背光闪烁的效果。

实测效果极佳，极大地改善了眼睛疼的问题。

语法：sct 3700 0.8（3700K，80%亮度遮罩）

高分屏适配

Gnome-Tweaks -> 字体 -> 缩放比例 1.10

能够满足高DPI缩放的需求，但是屏幕右上角会出现一个无障碍的小人……（强迫症贼难受）

目前未找到去掉无障碍小人图标的方法。

学习！

Anki 原生Linux版本完美适配！插件也都能完美使用！

部分网页版应用

经常会用到Microsoft To-Do 以及Outlook日历。Exchange协议的日历在Gnome-Calendar中的支持并不好（无法使用），无法绑定账户。因此改用网页版代替。

在/usr/share/applications下手动创建了图标，将两个网站的快捷方式加入到了Launchpad（是这样说吗）中，于是很愉快地放入了Dock。

多系统连通

VNC Connect实现公网ipv6直通访问宿舍电脑（几乎零延迟超好评！），FileZilla用于传输文件（平均3MB/s）
